#ctor

Member resides in the imsAO_AR_Application Class

Summary


This class handles two subdomains: Applications and Adjustments. Application lines can either be the primary line (created under the applying source doc) or a secondary lines (created under the applied to source doc). For a new row, which is which can be deduced by who the parent object is. If the parent is a ledger row then this row is a primary; if the parent is another application object then this row is a secondary.
