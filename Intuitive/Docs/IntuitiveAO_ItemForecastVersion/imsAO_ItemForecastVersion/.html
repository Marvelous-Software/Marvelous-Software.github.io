Member resides in the imsAO_ItemForecastVersion Class

Summary


This is a *virtual* AO. No database table exists for this app object. It's purpose is to allow the UI to set the item ID and forecast version ID that defines a set of IFL records that represent a particular version of the forecast for an item. Also, save and locking occur at this level.

Remarks


It should not be confused with AO_ForecastVersion which aligns with ItemForecastHeader table. For the record, the various AOs and their tables are: AO_ItemForecastVersion >> (no table) (parent, virtual) Domain = ItemForecastVersion AO_ItemForecast >>>>>>> ItemForecastLine table (child) Domain = ItemForecast AO_ForecastVersion >>>>> ItemForecastHeader table (join, for purpose of version name) Domain = ForecastVersion
