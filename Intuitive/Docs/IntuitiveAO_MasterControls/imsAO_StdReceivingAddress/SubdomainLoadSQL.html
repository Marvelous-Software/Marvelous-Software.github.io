SubdomainLoadSQL

Member resides in the imsAO_StdReceivingAddress Class

Summary


Allows the parent to specifiy the subdomain load SQL instead of the collection class. Overriding methods should not call the base method.

Remarks


The handle of the collection class is passed in to allow the parent to identify which subdomain it is. That would normally be the subdomain name or the logical subdomain name. So the code structure would be something like: dim cmd as new imsCommandObject Select Case collectionClass.Domain.ToLower Case "sub1" ... concoct cmd object Case "sub2" ... concoct cmd object End Select Return cmd

Returns


Nothing to indicate the collection class must concoct the SQL or an imsCommand object ready to go.
